Starten

Willkommen bei Aeropus! Diese Hilfeseite soll dir alle Funktionen unserer Plattform verständlich erklären. Egal ob du Links lokal verwaltest oder in der Cloud speicherst – hier findest du Schritt-für-Schritt-Anleitungen, Tipps zu erweiterten Funktionen und Lösungen für mögliche Probleme.

Die Oberfläche ist bewusst klar strukturiert: Header für den globalen Kontext, Sidebar für Navigation, Hauptbereich für Inhalte. So kannst du schnell zwischen Kapiteln und Funktionen wechseln.

Einführung in die Oberfläche

Die Aeropus-Oberfläche besteht aus drei Hauptbereichen:

  • Header: Zeigt den aktuellen Modus, Titel und ggf. Login-Status.
  • Sidebar: Enthält alle Kapitel und Funktionen, inklusive 🔒 geschützter Cloud-Funktionen.
  • Hauptbereich: Hier werden Inhalte, Formulare, QR-Codes und Anleitungen angezeigt.

Die Sidebar ist in Kapitel und Unterkapitel gegliedert, damit du jederzeit weißt, in welchem Abschnitt du dich befindest. Scrollst du durch die Seite, wird automatisch der aktuelle Punkt hervorgehoben.

Loslegen

Bevor du Cloudfunktionen nutzen kannst, musst du dich anmelden. Das sorgt dafür, dass deine Links sicher gespeichert werden. Für lokale Links ist keine Anmeldung nötig – du kannst direkt Dateien erstellen, bearbeiten oder öffnen.

Empfohlene Vorgehensweise:

  1. Cloudmodus: Anmeldung durchführen → Sidebar zeigt jetzt alle 🔒 Funktionen.
  2. Lokaler Modus: Datei auswählen oder neue .url08 erstellen → sofortiger Zugriff.
  3. QR-Code Vorschau prüfen → einfacher Zugriff via Smartphone.

Einen Link erstellen

Um einen neuen Link zu erstellen:

  • Cloudmodus: Cloudlink erstellen aufrufen → Formular ausfüllen.
  • Lokaler Modus: create.html → Name, URL, QR-Code Option, Passwort eingeben.

Tipps:

  • Wenn du QR-Code aktivierst, kannst du die URL direkt per Smartphone öffnen.
  • Für sensible Links empfiehlt sich Passwortschutz. Die Datei wird dann nur nach Eingabe eines 6-stelligen PINs geöffnet.
  • Speichere Cloud-Links regelmäßig, um Datenverlust zu vermeiden.

Einen Link bearbeiten

Bestehende Links können jederzeit bearbeitet werden:

  • Cloudmodus: Cloudlink bearbeiten → wähle den Link → bearbeite Name, URL, QR oder Passwort → Speichern.
  • Lokaler Modus: edit.html → Datei auswählen → Änderungen vornehmen.

Hinweise:

  • Passwortgeschützte Links erfordern die Eingabe der 6-stelligen PIN vor Bearbeitung.
  • Die Historie der Änderungen wird nicht automatisch gespeichert, daher ggf. lokale Sicherung anlegen.

Erweiterte Funktionen

Die Aeropus-Plattform bietet einige nützliche Erweiterungen:

  • Automatisches Speichern in der Cloud (Supabase).
  • QR-Code-Vorschau vor dem Öffnen oder Teilen.
  • OS-Warnungen, wenn Links auf iOS, alten Windows- oder macOS-Versionen geöffnet werden.
  • Forced Warnings per URL-Query (z.B. ?warning=applemobile).

Einen Link öffnen

Links können aus der Cloud oder lokal geöffnet werden:

  • Cloud: open-cloud.html → wähle gespeicherten Link → Passwort prüfen → QR-Code anzeigen.
  • Lokale Datei: open.html → .url oder .url08 auswählen → Passwort prüfen → QR-Code anzeigen.

Tipps:

  • Passwortgeschützte .url08 Dateien müssen die 6-stellige PIN eingeben, sonst kann der Inhalt nicht geöffnet werden.
  • Abbrechen ist möglich → Overlay schließt sich → Datei kann später erneut geöffnet werden.
  • „Zur Website navigieren“-Button öffnet die URL im neuen Tab.

URLs öffnen

Unterstützte Dateiformate:

  • .url: klassische Windows/Internet Shortcut Datei.
  • .url08: moderne Aeropus-Datei mit QR-Code und optionalem Passwort.

Empfehlungen:

  • Bei iOS/iPadOS immer .url08 verwenden → keine Kompatibilitätsprobleme.
  • Auf Windows 10+, macOS Big Sur+ oder Linux → sowohl .url als auch .url08 möglich.
  • QR-Codes erleichtern das schnelle Öffnen auf Mobilgeräten.

Sonderfall iOS

iOS und iPadOS erkennen klassische .url-Dateien nicht korrekt. Daher:

  • Beim Öffnen von .url erscheint ein Overlay mit Hinweis.
  • Empfohlene Lösung: erstelle .url08 Dateien mit Passwortschutz oder QR-Code.
  • Forced Warnings per URL-Query ermöglichen Entwicklern, Nutzer gezielt zu warnen.
  • So stellst du sicher, dass Links auf allen Plattformen korrekt geöffnet werden.

Einzelnachweise

Die Funktionen basieren auf Aeropus UI II, Supabase-Cloudintegration und moderner Browsertechnologie. QR-Codes werden via qrcode.js erzeugt. Die Passwortlogik für .url08 Dateien schützt Inhalte vor unbefugtem Zugriff. OS-Erkennung erfolgt über navigator.userAgent.

Weitere Informationen und Updates findest du in den Aeropus Release Notes und der offiziellen Dokumentation.
