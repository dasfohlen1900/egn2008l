Starten

Willkommen bei Aeropus! Diese Hilfeseite bietet detaillierte Erklärungen für alle Funktionen unserer Plattform, egal ob du Links lokal oder in der Cloud verwaltest. Schritt-für-Schritt-Anleitungen, Tipps zu erweiterten Funktionen und Lösungen für mögliche Probleme sind enthalten.

Die Oberfläche ist klar strukturiert: Header zeigt den Kontext, Sidebar die Navigation, Hauptbereich die Inhalte. So bleibst du jederzeit orientiert.

Einführung in die Oberfläche

Die Aeropus-Oberfläche besteht aus drei Hauptbereichen:

  • Header: Zeigt aktuellen Modus, Titel und Login-Status.
  • Sidebar: Enthält Kapitel und Funktionen, inklusive 🔒 geschützter Cloud-Funktionen.
  • Hauptbereich: Zeigt Inhalte, Formulare, QR-Codes und Anleitungen.

Die Sidebar bleibt fixiert am Rand und hebt beim Scrollen automatisch den aktuellen Punkt hervor.

Loslegen

Bevor du Cloudfunktionen nutzt, melde dich an. So werden Links sicher gespeichert. Lokale Links benötigen keine Anmeldung.

  1. Melde dich im Cloudmodus an → Sidebar zeigt 🔒 Funktionen.
  2. Für lokale Links → Datei auswählen oder neue .url08 erstellen → sofortiger Zugriff.
  3. QR-Code Vorschau prüfen → schneller Zugriff via Smartphone.

Einen Link erstellen

So erstellst du neue Links:

  • Cloudmodus: Cloudlink erstellen → Formular ausfüllen → Speichern.
  • Lokaler Modus: create.html → Name, URL, QR-Option, Passwort eingeben → Datei speichern.

Tipps:

  • QR-Code aktivieren für mobile Nutzung.
  • Passwortschutz schützt sensible Links (6-stelliger PIN).
  • Cloud-Links regelmäßig speichern.

Einen Link bearbeiten

  • Cloudmodus: Cloudlink bearbeiten → Link auswählen → Änderungen speichern.
  • Lokaler Modus: edit.html → Datei auswählen → Änderungen vornehmen.

Hinweise:

  • Passwortgeschützte Links benötigen PIN.
  • Lokale Historie nicht automatisch gespeichert → ggf. Backup erstellen.

Erweiterte Funktionen

  • Automatisches Speichern in Cloud (Supabase).
  • QR-Code-Vorschau vor Öffnen oder Teilen.
  • OS-Warnungen bei iOS, alten Windows/Mac-Versionen.
  • Forced Warnings per URL-Query möglich.

Einen Link öffnen

  • Cloud: open-cloud.html → Link auswählen → Passwort prüfen → QR-Code anzeigen.
  • Lokal: open.html → Datei auswählen → Passwort prüfen → QR-Code anzeigen.

Tipps:

  • Abbrechen schließt Overlay → Datei später erneut öffnen.
  • „Zur Website navigieren“-Button öffnet URL im neuen Tab.

URLs öffnen

Unterstützte Formate:

  • .url: klassische Windows/Internet Shortcut Datei.
  • .url08: moderne Aeropus-Datei mit QR-Code & Passwort.
  • iOS/iPadOS → nur .url08 verwenden.
  • Windows 10+, macOS Big Sur+ → beide Formate möglich.
  • QR-Codes für schnelle Mobilnutzung.

Sonderfall iOS

  • iOS/iPadOS erkennen klassische .url-Dateien nicht korrekt → Overlay erscheint.
  • Lösung: .url08 verwenden → Passwort oder QR-Code aktiviert.
  • Forced Warnings per URL-Query: ?warning=applemobile.

Einzelnachweise

Basis: Aeropus UI II, Supabase-Cloudintegration, moderne Browsertechnologien. QR-Codes via qrcode.js, Passwortlogik schützt .url08 Inhalte, OS-Erkennung über navigator.userAgent.
